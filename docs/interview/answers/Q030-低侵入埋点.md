# Q030: 什么是「低侵入埋点」？本项目提供了哪三种 adapter？各自适用什么场景？

## 问题

什么是「低侵入埋点」？本项目提供了哪三种 adapter？各自适用什么场景？

## 参考答案

低侵入埋点是指在不大幅修改现有 agent 代码的前提下，通过包装器或回调机制自动采集 agent 执行轨迹。项目在 `sdk/adapters/` 下提供了三种 adapter，按侵入程度从低到高排列。

**1. LangGraph Adapter（推荐，1 行代码）。** `sdk/adapters/langgraph.py:378-410` 的 `instrument_langgraph()` 工厂函数接收一个 `StateGraph` 或 `CompiledStateGraph`，返回一个透明包装器 `InstrumentedStateGraph`（line 66-309）。它在初始化时自动遍历图中所有节点（line 87-91），将每个节点的 `runnable` 替换为带有轨迹记录逻辑的包装版本（line 93-103）。后续调用 `add_node` 也会自动包装新节点（line 105-109）。使用方式：

```python
from sdk.adapters.langgraph import instrument_langgraph
graph = instrument_langgraph(build_graph())
result = await graph.ainvoke(initial_state)
```

**2. LLM Proxy（1 行代码）。** `sdk/adapters/llm_proxy.py:42-237` 的 `ProxyChatModel` 继承自 `BaseChatModel`，包装原始 LLM 实例。它在 `_generate`（line 64-79）和 `_agenerate` 方法中拦截调用，记录输入消息、输出结果和耗时，然后转发给原始 LLM。工厂函数 `create_proxy_llm()`（line 240-266）负责创建代理。使用方式：

```python
from sdk.adapters.llm_proxy import create_proxy_llm
llm = create_proxy_llm(ChatZhipuAI(...))
response = llm.invoke("Hello")  # 使用方式完全不变
```

**3. Callback Handler（挂载到 chain/LLM）。** `sdk/adapters/callback.py:44-285` 的 `EvalCallbackHandler` 实现 LangChain 的 `BaseCallbackHandler` 接口，捕获 `on_llm_start`（line 63-76）、`on_chat_model_start`（line 78-89）、`on_tool_start`/`on_tool_end`、`on_chain_start`/`on_chain_end` 等细粒度事件。工厂函数 `create_callback_handler()`（line 288-307）创建实例。使用方式：

```python
from sdk.adapters.callback import create_callback_handler
handler = create_callback_handler()
llm = ChatZhipuAI(callbacks=[handler])
```

**选择建议：**
- LangGraph-based agent：优先用 `instrument_langgraph`，自动覆盖所有节点，无需逐个包装。
- 只需要记录 LLM 调用：用 `create_proxy_llm`，对非 LangGraph 的 agent 也适用。
- 需要细粒度事件（如 `on_llm_start` 的 prompt 级别捕获）或非图结构的 LangChain agent：用 `create_callback_handler`。

三种 adapter 都将轨迹记录到同一个 `TrajectoryCollector` 单例（通过 `get_collector()` 获取），因此可以混用。所有工厂函数在 `sdk/adapters/__init__.py:5-7` 统一导出。

## 代码依据

- `sdk/adapters/langgraph.py:66-309` — InstrumentedStateGraph 透明包装 StateGraph，自动包装所有节点
- `sdk/adapters/langgraph.py:378-410` — `instrument_langgraph()` 工厂函数，1 行代码完成接入
- `sdk/adapters/llm_proxy.py:42-237` — ProxyChatModel 继承 BaseChatModel，拦截 generate/agenerate
- `sdk/adapters/llm_proxy.py:240-266` — `create_proxy_llm()` 工厂函数
- `sdk/adapters/callback.py:44-285` — EvalCallbackHandler 实现细粒度 LangChain 回调
- `sdk/adapters/callback.py:288-307` — `create_callback_handler()` 工厂函数
- `sdk/adapters/__init__.py:5-7` — 统一导出三个工厂函数

## 回答要点

- 低侵入 = 最小代码改动，通过包装器/回调自动采集轨迹
- LangGraph adapter：1 行代码，自动包装图中所有节点，推荐用于 LangGraph agent
- LLM Proxy：1 行代码，透明包装 LLM 调用，适合只需记录 LLM 输入输出的场景
- Callback：挂载到 LangChain LLM/chain，捕获细粒度事件（on_llm_start 等）
- 三种 adapter 都写入同一个 TrajectoryCollector 单例，可混用

## 常见追问

**Q: 三种 adapter 可以同时使用吗？**

A: 可以。它们都调用同一个 `get_collector()` 获取全局 `TrajectoryCollector` 单例，轨迹会被合并到同一个 task 的 step 列表中。例如 LangGraph adapter 记录节点级轨迹，同时 LLM Proxy 记录 LLM 调用细节。

**Q: `InstrumentedStateGraph` 如何处理图编译后的节点？**

A: 如果传入的是 `CompiledStateGraph`，工厂函数（langgraph.py:401-402）会返回 `InstrumentedCompiledGraph`，它在编译后的图上做同样的包装。未编译的 `StateGraph` 则返回 `InstrumentedStateGraph`，在 `compile()` 之前完成节点包装。
