## 问题

如果第三方 Agent 框架（如 Semantic Kernel）无法按你的 schema 上报轨迹，你会怎么适配？

## 参考答案

第三方框架适配策略：1) 手动 sdk/collector record_*；2) 写新 adapter 映射到 ActionType；3) 离线转换 batch 导入 POST tasks/steps。Semantic Kernel 等可包装 step 回调→统一 JSON schema。关键是 tool_call/tool_result 配对与 retrieval 的 retrieved_docs 字段对齐 app/models/schemas.py TrajectoryStep。

## 代码依据

- `sdk/collector.py`
- `sdk/adapters/callback.py`
- `app/models/schemas.py`

## 回答要点

- adapter 模式扩展框架
- 手动 collector 最低成本 PoC
- schema 对齐比框架更重要
- 可批量 ETL 历史 log

## 常见追问

**Q: 需要改平台吗？**

A: 通常只加 adapter，不改 Evaluator。

**Q: 最小字段集？**

A: step_number、action_type、action_detail、observation。
