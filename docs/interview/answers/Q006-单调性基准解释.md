## 问题

如果让你向非技术人员（产品经理）解释「单调性基准测试」，你会怎么说？

## 参考答案

向产品经理可这样说：我们准备了六档「假 Agent 行为剧本」（优秀到空轨迹），跑同一套打分器，分数应该越差越低；若中等比良好还高，说明打分器有问题。容差 0.05 允许 LLM Judge 小幅波动。通过标准在 monotonicity.py 的 check_monotonicity：沿 QUALITY_ORDER  overall 非递增（允许 +0.05）。REFERENCE_SCORES 约 93.1→20.0 是历史标定参考。



> 单调性 Synthetic Benchmark 是用于验证评估体系可靠性的基准测试。我们人工构造一组质量逐渐退化的 Agent 轨迹，例如逐步减少 Planning 覆盖率、增加 Tool Use 错误、降低 Retrieval 质量、移除 Replan 行为等，从而形成一组已知质量排序的轨迹。然后运行六维评估器，检查最终得分是否能够保持单调下降。如果高质量轨迹持续获得高分、低质量轨迹持续获得低分，则说明评估器具有良好的区分能力和排序能力。它本质上是在评估 Evaluator 本身，而不是评估 Agent，因此可以作为评估系统的回归测试和可靠性验证基准。

## 回答要点

- 六条合成轨迹对应质量档位
- 分数应单调下降，容差 +0.05
- 用于 CI 门禁与 prompt 变更回归
- REFERENCE_SCORES 为标定参考非硬阈值

## 常见追问

**Q: 为什么要容差？**

A: LLM Judge 有随机性边界，严格单调易误报。

**Q: 失败了怎么办？**

A: 看各维 dim_scores 哪一档逆序，定位具体 Evaluator。
