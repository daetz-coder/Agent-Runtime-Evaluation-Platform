## 问题

平台评估的是 Agent 的「过程质量」还是「结果质量」？两者如何平衡？

## 参考答案

平台同时评估过程与结果：Tactical/Tool Use 看每步决策与工具使用，Retrieval 看证据是否支撑回答，Planning 看计划覆盖与粒度；overall_score 在 evaluation_graph.py 加权汇总，也生成 summary 与 recommendations。Trajectory 中的 observation 与 failure 反映结果侧信号。非 RAG Agent 的 Retrieval 可能为 0，需在解读 overall 时看维度适用性。Wiki Demo 中 search 节点 record_retrieval 把检索过程显式写入轨迹供 RetrievalEvaluator 消费。

## 代码依据

- `app/graphs/evaluation_graph.py`
- `app/evaluators/tactical_evaluator.py`
- `app/evaluators/retrieval_evaluator.py`

## 回答要点

- 过程：plan/tool/replan/retrieval 动作链
- 结果：failure、最终 observation、goal 达成隐含在 Judge prompt
- aggregate 生成 summary + recommendations
- 解读 overall 需看 Agent 类型是否适用六维

## 常见追问

**Q: 能否只评结果？**

A: 可以弱化过程维权重，但会失去定位根因的能力。

**Q: 如何平衡？**

A: RAG Agent 提高 retrieval 权重；工具型 Agent 提高 tool_use。
